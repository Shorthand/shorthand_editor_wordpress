== Installation ==

= Minimum Requirements =

- WordPress 6.0 or greater
- PHP version 7.2 or greater
- HTTPS support

= Get the Plugin =

The Shorthand Editor is distributed as a ZIP file. The latest version may
be downloaded from the
[Shorthand website](https://www.shorthand.com/shorthandforwordpress/).

= Install the Plugin =

You can follow the instructions provided on the WordPress codex relating to
[Uploading](https://wordpress.org/documentation/article/manage-plugins/#upload-via-wordpress-admin)
or [Manual installation](https://wordpress.org/documentation/article/manage-plugins/#manual-plugin-installation-1)
to install it on your WordPress instance.

Once you have installed and activated the plugin, don't forget to follow
the instructions to [Connect to Shorthand](#connect-to-shorthand)!

= Connect to Shorthand =

Once the plugin has been installed, you will need to connect your WordPress site
to Shorthand. Only a WordPress user with the ability to activate plugins can perform
this task.

During this process, you will establish a Shorthand account and workspace
for the content you will create.

1. Navigate the dashboard to the "Installed Plugins" page in the Plugins menu
1. Find the plugin named "The Shorthand Editor" and click "Connect to Shorthand…"
1. Sign in with your Shorthand account, or create a new account by selecting "Start Creating For Free"
1. Select a new or existing workspace to be associated with your organisation
1. Select a new or existing WordPress team for your site
1. Click "Connect" to complete the connection

== Frequently Asked Questions ==

**Does The Shorthand Editor plugin use an external service?**

Yes! The Shorthand Editor plugin brings WordPress and the Shorthand platform together, to allow you to create and edit rich,
engaging stories in the Shorthand editor, preview them as web pages on your WordPress dashboard, and publish them to your
WordPress hosting.

**How does The Shorthand Editor plugin access Shorthand, specifically?**

During the course of operation, a user may direct this plugin to

- associate a WordPress site with a Shorthand workspace
- authorise a user to access Shorthand content
- navigate a user to an authorised page on the Shorthand website to create or edit their story content
- enumerate, access and update the story content created using the interactive story editor
- obtain previews of story content for display on the admin dashboard when editing or previewing a story
- download the CSS, JS, HTML, and other asset files of a story that a user publishes to WordPress

To provide this functionality, the plugin may navigate to the Shorthand web application in the browser at
[app.shorthand.com](https://app.shorthand.com), or make requests of the Shorthand API at
[api.shorthand.com](https://api.shorthand.com) from the WordPress instance.

A published story does not require the Shorthand platform, although it may load code from Shorthand in order to
facilitate embedded rich content, or to provide anonymised page view counts. These scripts may be served from the
[iframely.shorthand.com](https://iframely.shorthand.com) and
[analytics.shorthand.com](https://analytics.shorthand.com) domains respectively.

This service is provided by Shorthand Pty Ltd, and is covered by these [terms of use](https://shorthand.com/legal/customer-terms/)
and [privacy policy](https://shorthand.com/legal/privacy-policy/).

**Is Shorthand a page builder?**

Not exactly. Shorthand is a **visual storytelling platform** designed for premium editorial content—not for building full site layouts.

**Will Shorthand slow down my WordPress site?**

No. Shorthand is built for **speed, optimization, and minimal footprint**. It avoids the bloat typical of large page builders.

**Do I need a paid account to use The Shorthand Editor plugin?**

The Shorthand Editor must be [connected](#connect-to-shorthand) to a Shorthand workspace, where you can can create and edit stories
for free. When you are ready to publish, activate your subscription to obtains credits for publishing. (Compare [Shorthand plans here](https://shorthand.com/pricing/).)

Once your stories have been published, they reside on your WordPress site.

**Does Shorthand work with my theme?**

Yes. You can craft bespoke custom themes in Shorthand to standardize colors, fonts, logos, buttons, whole sections and more across stories.

**Is this suitable for enterprise environments?**

Yes. With ISO27001/SOC2 compliance, WordPress-native permissions, and secure infrastructure, Shorthand is built for enterprise teams.

**Can I use Shorthand with Gutenberg?**

Yes. Shorthand complements Gutenberg by providing a specialized tool for rich, scroll-based narratives.

**Where can I find the source code?**

You can find the source code for this plugin at [The Shorthand Editor](https://github.com/shorthand/shorthand_editor_wordpress).

== Troubleshooting ==

In general, we recommend the following:

- re-activating WP-Cron if it is not enabled
- checking that the Shorthand Connect plugin has been deactivated

If you are experiencing issues when navigating between Shorthand and WordPress, try

- ensuring that your WordPress site is configured to use HTTPS; and
- following the process to [reconnect to your existing WordPress team in Shorthand](#connect-to-shorthand).

If you are experiencing publishing issues, ensure that your Shorthand workspace has been
activated, and has sufficient story credit.

Some WordPress plugins can aggressively cache or control how content is displayed at
the theme-level. If a Shorthand story is not displaying correctly when published, try
different WordPress themes, or configure additional CSS under the plugin's settings
page.

For further support and feedback, contact help@shorthand.com.
